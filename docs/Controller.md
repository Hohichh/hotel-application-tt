# Controller layer design
---

Для начала, поскольку все префиксы имеют общий маршрут - стоит внести его в .properties

Мне кажется имеет смысл сделать 2 контроллера - 1 для гистограмм,другой для отелей. 
В первом будут только методы для получения гистограмм, во втором - все остальные. 

В теории методы группировки использовались бы для статистики/аналитики что потребовало бы отдельных фильтров безопасности/доступа. Судя по корню histogram. 

В остальном маршруты как указанаы в тз. Помечай их аннотациями в завиисмости от типа метода запроса. Указвый valid а также pathvariable и requestparam. Контроллеры будут типа RestController, тк это API. Контроллеры агрегируют сервис. Отправляют ResponseEntity с нужным статусом.
