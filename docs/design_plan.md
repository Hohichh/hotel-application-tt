# Этапы проектирования приложения Hotel Management API

На основе технического задания определены следующие этапы проектирования:

- **Определение модели данных на основе JSON-контрактов**: 
  - Проанализировать структуру ответов эндпоинтов `GET /hotels` и `GET /hotels/{id}`.
  - Спроектировать сущность `Hotel` и связанные с ней компоненты (`Address`, `Contacts`, `ArrivalTime`).
  - Разработать схему хранения `amenities` (список строк), учитывая нормальные формы (выделение в отдельную таблицу с отношением Many-to-Many или One-to-Many).

- **Разработка UML-диаграммы базы данных**:
  - Описать таблицы и их связи (например, `Hotel`, `Amenity`, `Hotel_Amenity`).
  - Обеспечить соответствие нормальным формам для минимизации избыточности.

- **Проектирование архитектуры слоев (Class Diagram)**:
  - Выделить слои: `Controller`, `Service`, `Repository`, `Entity`, `DTO`.
  - Определить интерфейсы для сервисов и репозиториев для обеспечения расширяемости.
  - Предусмотреть маппинг между сущностями БД и DTO (использование MapStruct или аналогичных инструментов).

- **Проектирование механизмов поиска и фильтрации**:
  - Определить логику обработки параметров в `GET /search` (name, brand, city, country, amenities).
  - Спроектировать использование Spring Data JPA Specifications или Querydsl для динамических запросов.

- **Проектирование эндпоинтов статистики (Histogram)**:
  - Определить логику агрегации данных для `GET /histogram/{param}`.
  - Рассмотреть использование кастомных JPQL/Native запросов для эффективного подсчета.

- **Настройка управления схемой БД (Liquibase)**:
  - Подготовить changelog-файлы для создания таблиц и начальных данных.

- **Интеграция документации и тестирования**:
  - Добавить поддержку Swagger/OpenAPI (SpringDoc).
  - Определить стратегию тестирования (Unit-тесты для сервисов, Integration-тесты с использованием `@SpringBootTest` и H2).

- **Обеспечение гибкости конфигурации**:
  - Вынести настройки БД и порта (8092) в `application.properties`.
  - Предусмотреть возможность переключения профилей БД.
